# Доработка фрагмента Markdown

## Когда применять

- В чате есть **выделенный фрагмент** из `.md` и запрос на пояснение или переписывание.
- Вызвана команда `/refine-md`.

## Workflow

1. Взять выделение из сообщения пользователя.
2. Прочитать исходный файл целиком (путь из контекста или открытый редактор).
3. Диагностика: 1–3 пункта — что не так.
4. Переписать фрагмент по MDS и правилам стиля; терминология — `02_Терминология.mdc`.
5. Применить правку в **исходном** `.md` (StrReplace / edit), не в preview.

## Не делать

- Не дублировать правила MDS в ответе.
- Не трогать `reference/`.
- Не менять структуру документа без запроса — только фрагмент.

## Связь

Slash-команда: `.cursor/commands/refine-md.md`
