# Проектирование урока

## Роль

Один урок — одна **центральная идея** (MDS §4). Урок закрывает пробел в способностях, не «проходит тему».

## Обязательные документы

| Документ | Зачем |
|---|---|
| `docs/02_MATERIAL_DESIGN_STANDARD.md` | правила, 10 вопросов, решения учащегося |
| `docs/04_LESSON_DESIGN.md` | шаблон LESSON.md |
| `docs/01_PEDAGOGY.md` §7, §11 | роль преподавателя, выбор формата |
| `modules/.../UNIT.md` | место урока в модуле |

Форматы: `docs/05_NOTEBOOK_STANDARD.md`, `docs/06_PRESENTATION_STANDARD.md`.

Эталон: `modules/08_functions_recursion/lessons/01_function_as_mapping/`.

## Куда класть результат

```
modules/<module>/lessons/<NN>_<slug>/
  LESSON.md       # заполненный Lesson Design
  lesson.ipynb    # если нужен ноутбук
  slides.md       # если нужна презентация (Markdown-слайды)
  homework_*      # опционально
```

## Workflow

```
- [ ] 1. Пройти 10 вопросов MDS §2 для этого урока
- [ ] 2. Заполнить LESSON.md (разделы 1–12)
- [ ] 3. Одна центральная идея — проверить таблицу §3
- [ ] 4. Минимум 2 решения учащегося (§7) — иначе перепроектировать
- [ ] 5. Выбор формата (§10): ноутбук / презентация / только код — обосновать
- [ ] 6. Ожидаемые ошибки (§9) — конкретные, с обратной связью
- [ ] 7. Создать материалы только отмеченные в §10
- [ ] 8. Чек-лист Lesson Design §12 + MDS §11
```

## Ноутбук (если нужен)

Структура по Notebook Standard §2:

1. цель и data/ML-контекст (если уместно);
2. результаты обучения;
3. пререквизиты;
4. короткие ячейки, ≥1 эксперимент без готового ответа;
5. самостоятельное задание — **расширение**, не повтор;
6. импорт сквозных данных из `data/module_datasets.py`, если модуль их использует.

Пересборка ipynb: `modules/<module>/generate_notebooks.py` — править скрипт, не json вручную.

## Презентация (если нужна)

Только когда нужно **общее визуальное внимание** (Presentation Standard). Не дублировать ноутбук. Формат: `slides.md`, один слайд — одна цель.

## Data/ML

- Профессиональный контекст в §5 — конкретный (sklearn API, метрика, pipeline), не общие слова.
- Примеры на данных модуля, не абстрактные `x`, `y`, если есть `data/module_datasets.py`.

## Правила

- Не дублировать правила MDS в LESSON.md — ссылки на разделы.
- Домашнее задание: только если развивает способность, которую нельзя закрыть на уроке (§11).
- Не выходить за «Вне scope» модуля (UNIT.md §12).

## После завершения

Сообщить: центральная идея, какие материалы созданы, какой артефакт урока, связь с модулем.
