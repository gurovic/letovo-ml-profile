# Проектирование модуля

## Роль

Инженер документации, не автор учебника. Модуль — решение обучающей проблемы, не список тем.

## Обязательные документы (читать перед работой)

| Документ | Зачем |
|---|---|
| `docs/00_FOUNDATION.md` §7 | траектория класса |
| `docs/01_PEDAGOGY.md` §5, §11 | последовательность, выбор формата |
| `docs/02_MATERIAL_DESIGN_STANDARD.md` §2, §11 | 10 вопросов, чек-лист |
| `docs/03_UNIT_PLANNER.md` | шаблон UNIT.md |

Эталон модуля (глубина UNIT + уроков + ноутбуков): `modules/08_01_functions_recursion/`.  
Тонкий scaffold «чтобы закрыть КТП/Canvas» — **не** готовый модуль курса.

## Куда класть результат

```
modules/<grade>_<nn>_<slug>/
  README.md          # кратко: цель, уроки, запуск
  UNIT.md            # заполненный Unit Planner
  data/              # опционально: сквозные датасеты модуля
  lessons/           # папки уроков (Lesson Design — отдельный skill)
  artifact/          # итоговый артефакт модуля
```

Имена: латиница, snake_case, префикс `класс_номер_` (`08_01_`, `08_02_`, …; для 9 класса — `09_01_` и т.д.). Номер — порядок модуля в КТП года.

## Workflow

```
- [ ] 1. Пройти 10 вопросов MDS §2 — если нет ответа, не писать материалы
- [ ] 2. Сверить с Foundation §7 (класс, пререквизиты, переход)
- [ ] 3. Заполнить UNIT.md полностью (разделы 1–19); §17 — рефлексия автора; §18 — глоссарий модуля
- [ ] 4. Одна главная идея модуля — иначе разбить на два
- [ ] 5. Таблица уроков: зачем, центральная идея, активность, артефакт; роль **введение / отработка** для кода; у каждого урока — `LESSON.md` §13
- [ ] 6. Итоговый артефакт модуля — проверяемый, не «я понял»; опирается на уже отработанные навыки
- [ ] 7. «Вне scope» — явно перечислить
- [ ] 8. Чек-лист Unit Planner §19 + MDS §11; для Python/алгоритмов — пары практики в плане (не одно упражнение)
- [ ] 9. Строки [матрицы навыков](../../docs/ktp/08_skills_matrix.md) для пар модуля — заполнены или обновлены ([07_KTP](../../docs/07_KTP.md))
```

## Data/ML в программном модуле

Если модуль про Python/алгоритмы в профиле ML:

1. **Сквозной микро-датасет** в `data/module_datasets.py` (списки/словари, без pandas).
2. Три линии: **describe → transform → predict**.
3. Связь с будущими модулями (EDA, sklearn) — одним абзацем в UNIT.md, не дублировать их содержание.

## Правила

- Не дублировать MDS в UNIT.md — только ссылки.
- Не создавать уроки/ноутбуки в этом skill — только план; материалы → skill `design-lesson` (с воротами глубины ноутбука).
- Модуль «готов» только если уроки прошли `design-lesson` до публикации: stubs+assert, полные `solutions`, не прогон готовых ответов ([Notebook Standard §3](../../docs/05_NOTEBOOK_STANDARD.md)).
- При изменении, затрагивающем Foundation/Pedagogy — **сообщить пользователю**.
- Терминология: `.cursor/rules/02_Терминология.mdc`.
- Язык материалов: русский; в коде и chat — без смешения кириллицы и латиницы в одном слове.

## Антипаттерны (Foundation §10)

- модуль = гонка тем без артефакта;
- урок «по традиции»;
- ML-мотивация без data-активности учащегося.

## После завершения

Кратко сообщить: цель модуля, число уроков, артефакт, что сознательно вне scope.

Рекомендуется прогон skill `test-teacher` по `UNIT.md` и ключевым `LESSON.md` перед формальным `review-edu-material`.
