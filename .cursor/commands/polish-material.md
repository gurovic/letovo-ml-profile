# Довести материал до качества (редактор-эксперт)

Запустить skill **expert-edu-editor**: итеративно править материалы или **вернуть на переработку**, если халтура.

## Scope

Уточнить у пользователя, если не указано: модуль, урок, список файлов.

## Ворота (обязательны)

1. Педагогика — MDS, деятельность, одна идея, артефакт.
2. Наука/математика — корректность, уровень класса.
3. Инженерия — практики, код, профконтекст.
4. Ясность для учителя — каждое слово однозначно.
5. Без забегания вперёд — пререквизиты и КТП.
6. Уровень одарённых — глубина, без занижения.
7. **Без воды** — вычистить пустые и повторные фразы.
8. **Целостность** — UNIT, LESSON, ноутбук, КТП, §13 согласованы.

## Халтура

При признаках формальной заготовки — вердикт **перепроектировать**, не косметить. Запустить design-module / design-lesson.

## Действия

1. Аудит → правки **или** отчёт о переработке.
2. Повторять цикл, пока нет критичных/важных замечаний.
3. Отчёт с вердиктом.

Не писать в `reference/`.
