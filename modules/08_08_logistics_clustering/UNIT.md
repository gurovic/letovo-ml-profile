## 2. Зачем существует модуль

**Проблема:** таблицы pandas не объясняют стек, очередь и hash map; без структур нет эффективного preprocessing и подсчёта частот; кластеризация остаётся «чёрным ящиком».

**Если убрать модуль:** dict как hash map не формализован; аномалии в логистике не связаны с плотностью и расстоянием.

## 5. Результаты обучения

После модуля учащийся может:

1. **Применить** стек для отмены шагов preprocessing.
2. **Использовать** очередь и `deque` как буфер событий.
3. **Применить** `set` для уникальных id и проверки membership.
4. **Использовать** `dict` как hash map для частот и счётчиков.
5. **Объяснить** идею k-means: кластеры в пространстве признаков.
6. **Объяснить** DBSCAN: плотность и выбросы.
7. **Определить** аномалию как редкое или далёкое наблюдение.
8. **Найти** аномалии и сегменты опозданий на табличных данных Olist (slim).

## 18. Глоссарий модуля

| Термин | Определение в контексте модуля |
|---|---|
| Стек (stack) | LIFO: последний добавленный извлекается первым |
| Очередь (queue) | FIFO: первый добавленный извлекается первым |
| `deque` | Двусторонняя очередь с быстрыми операциями с концов |
| Hash map (`dict`) | Ключ → значение с быстрым доступом по ключу |
| k-means | Разбиение на k кластеров минимизацией расстояния до центроидов |
| DBSCAN | Кластеризация по плотности; точки вне плотности — шум |
| Аномалия | Наблюдение, сильно отличающееся от большинства |
