## 2. Зачем существует модуль

**Проблема:** ML на таблицах не заменяет работу с упорядоченными логами в памяти; без поиска и сортировки нет интуиции сложности и выбора алгоритма.

**Если убрать модуль:** бинарный поиск и mergesort остаются абстракциями; модуль 8 (структуры) теряет контекст отсортированных данных.

## 5. Результаты обучения

После модуля учащийся может:

1. **Реализовать** линейный поиск в массиве событий.
2. **Реализовать** бинарный поиск на отсортированном массиве и **оценить** сложность.
3. **Реализовать** сортировку выбором и **объяснить** O(n²).
4. **Реализовать** mergesort рекурсивно.
5. **Описать** quicksort (средний случай) на уровне идеи.
6. **Применить** `sorted(key=)` для сортировки записей лога.
7. **Решить** задачу двумя указателями на отсортированных данных.
8. **Сравнить** время O(n²) и O(n log n) на эксперименте.
9. **Применить** `binary_search` (и при наличии — `max_digit_recursive`) к логам; Canvas-задачи — в практике поиска.

## 18. Глоссарий модуля

| Термин | Определение в контексте модуля |
|---|---|
| Линейный поиск | Проверка каждого элемента до совпадения |
| Бинарный поиск | Деление отсортированного диапазона пополам |
| Сортировка выбором | На каждом шаге выбор минимума в неотсортированной части |
| Mergesort | Разделить → отсортировать → слить два отсортированных |
| Quicksort | Разбиение по опорному элементу (pivot) |
| Два указателя | Два индекса, движущиеся по массиву по правилам |
| O(n²) / O(n log n) | Качественная оценка роста времени |
