# Артефакт модуля: EDA-отчёт для исторического общества

**Тип:** Markdown-отчёт + папка графиков  
**Срок:** пара КТП **23**  
**Файл сдачи:** заполненный `report.md` (или эквивалент в ноутбуке) + `figures/*.png`  
**На паре 23:** сборка и сдача в `lessons/07_eda_report/`  

**Смысл для класса:** ответить «кто выживал чаще?» **без** обученной модели.

Данные: [data/titanic.csv](../data/titanic.csv) — стандартный Titanic (`seaborn.load_dataset('titanic')`).

Starter: [starter/README.md](starter/README.md)

---

## Задача

Подготовить отчёт для «исторического общества»:

1. описать источник и размер таблицы;
2. привести доли выживших overall и по ключевым группам (sex, pclass; желательно who);
3. показать описательную статистику age/fare и ≥2 графика;
4. зафиксировать пропуски и выбранную стратегию (что не делали — тоже);
5. назвать ограничения выборки / sampling bias;
6. дать вывод **без** `fit` / sklearn и **без** необоснованной причинности.

---

## Критерии приёмки

| # | Критерий |
|---|---|
| 1 | Указан источник (`titanic.csv` / seaborn Titanic), 891 строка |
| 2 | Есть числа: overall + ≥2 групповые доли survived |
| 3 | Есть describe или явные mean/median/std для age или fare |
| 4 | ≥2 графика в `figures/` с отсылкой в тексте |
| 5 | Раздел пропусков (age, deck, embarked) и решение по чистке |
| 6 | Раздел ограничений / bias |
| 7 | Вывод согласован с графиками; нет обучения модели |
| 8 | Нет формулировок «X является причиной Y» без оговорки |

## Решения учащегося (зафиксировать в отчёте)

1. Какие группы сравнивать в «заголовке» для общества.
2. Политика по `age` / `deck` (drop / median / не использовать).
3. Как сформулировать вывод без причинности.
