# Lesson Design: Лямбда-функции

## 1. Идентификация

| Поле | Значение |
|---|---|
| Модуль | Функции и рекурсия в Python |
| Название урока | Lambda |
| Длительность | 2 академических часа |
| Пререквизиты | Уроки 1–4; map, filter, sorted |

---

## 2. Зачем существует урок

Короткие transform на данных удобно передавать в `map`/`filter`/`sorted(key=)` — тот же приём, что `.apply(lambda)` в pandas (модуль 2 по КТП).

---

## 3. Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | Lambda — анонимная функция для одноразового transform в HOF |
| Что поддерживает | нормализация, фильтр аномалий, ранжирование моделей |

**Данные:** `EXAM_SCORES`, `FEATURE_ROWS`, `MODEL_RUNS`, `FEATURE_POINTS` ([data/module_datasets.py](../../data/module_datasets.py))

---

## 4. Результаты обучения

1. **Нормализовать** scores через `map` + lambda.
2. **Отфильтровать** аномалии по порогу.
3. **Отсортировать** модели по `f1` с `key=`.
4. **Построить** leaderboard `MODEL_RUNS` по f1 ↓, id ↑.
5. **Найти** farthest point в `FEATURE_POINTS` через `max(..., key=)` (Canvas).

---

## 5. Профессиональный контекст

pandas `.apply(lambda row: ...)` — тот же паттерн на таблицах; здесь — на списках и dict.

---

## 6. Структура урока

| Этап | Цель | Активность |
|---|---|---|
| map | Нормализация | lambda в map |
| filter | Аномалии | порог на EXAM_SCORES |
| sorted | Ранжирование | key по метрике |
| Конкурс | Tie-break | leaderboard MODEL_RUNS |
| Углубление | Расстояние | farthest point |
