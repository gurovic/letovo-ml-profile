## 2. Зачем существует урок

Закрывает модуль **сдачей**: сравнение классов текстов, наивная классификация, README ученика и рефлексия — проверяемый итог 11 пар.

---

## 3. Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | Готовый артефакт = 10 зелёных тестов + README с решениями ученика |
| Что поддерживает, но не отвлекает | Порог `ratio`; tie-break → `unknown`; рефлексия модуля |
| Данные урока | `TEXTS_POSITIVE`, `TEXTS_NEGATIVE` |

---

## 4. Результаты обучения

1. **Реализовать** `compare_class_frequencies` и `naive_classify` (шаги 6–8).
2. **Пройти** все 10 тестов `manual_tests.py`.
3. **Написать** README: 5 своих фраз для `naive_classify` (≥4 верных), pipeline, tie-break, связь с NLP.
4. **Сформулировать** краткую рефлексию модуля (что умеет после 11 пар).

---

## 5. Профессиональный контекст

Сдача по тестам + краткий отчёт — норма инженерного цикла до «модели в sklearn».

## 9. Ожидаемые ошибки

| Тип | Что делает / думает учащийся | Что говорит или показывает учитель |
|---|---|---|
| ML | `naive_classify` всегда `unknown` | Сначала проверить `compare_class_frequencies`; tie → unknown |
| NLP | Мало маркеров | Нужны рабочие tokenize / aggregate |
| Организация | Нет своих 5 фраз в README | Критерий приёмки №3 |
