# Lesson Design: Функции высшего порядка

## 1. Идентификация

| Поле | Значение |
|---|---|
| Модуль | Функции и рекурсия в Python |
| Название урока | HOF и inference pipeline |
| Длительность | 2 академических часа |
| Пререквизиты | Уроки 1–5; функции transform и predict |

---

## 2. Зачем существует урок

Отдельные функции нужно **собрать в цепочку** с фиксированным порядком шагов — прототип sklearn `Pipeline` без sklearn.

---

## 3. Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | HOF `apply_pipeline` выполняет цепочку transform → predict на одном объекте |
| Что поддерживает | extract → scale → predict; текстовый pipeline (задел) |

**Данные:** `FEATURE_ROWS`, `APARTMENTS`, тексты ([data/module_datasets.py](../../data/module_datasets.py))

---

## 4. Результаты обучения

1. **Реализовать** `apply_pipeline(data, steps)`.
2. **Собрать** pipeline для одной строки `FEATURE_ROWS`.
3. **Реализовать** `compose(f, g)` для двух шагов.
4. **Объяснить** соответствие списку `steps` в sklearn `Pipeline` (концепт, без fit).
5. **Собрать** текстовый pipeline (strip → split → filter tokens) как задел к `text_stats`.

---

## 5. Профессиональный контекст

sklearn `Pipeline(steps=[...])` — тот же порядок: preprocessing, затем модель. `fit` — в следующих модулях.

---

## 6. Структура урока

| Этап | Цель | Активность |
|---|---|---|
| apply_pipeline | Цепочка шагов | код HOF |
| Табличный pipeline | extract → scale → predict | FEATURE_ROWS |
| compose | Композиция двух функций | compose(f, g) |
| Текст | Задел NLP | lambda-шаги на строке |
| Связь | sklearn | схема без импорта |

---

---

## 9. Ожидаемые ошибки

| Тип | Ошибка | Обратная связь |
|---|---|---|
| Pipeline | Неверный порядок transform | Прогнать одну строку по шагам |
| Тип | Шаг возвращает не тот тип | Проверить выход каждого шага |
