# Lesson Design: Рекурсия

## 1. Идентификация

| Поле | Значение |
|---|---|
| Модуль | Функции и рекурсия в Python |
| Название урока | Рекурсия |
| Длительность | 2 академических часа |
| Пререквизиты | Уроки 1–3; функции, return |

---

## 2. Зачем существует урок

Вложенные ответы API и деревья категорий **не разбираются одним циклом**. Рекурсия — инструмент для таких структур (задел под decision tree).

---

## 3. Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | Рекурсия с базовым случаем обходит вложенные данные и деревья |
| Что поддерживает | flatten, extract_ids, walk_categories |

**Данные:** `NESTED_API_RESPONSE`, `CATEGORY_TREE` ([data/module_datasets.py](../../data/module_datasets.py))

---

## 4. Результаты обучения

1. **Реализовать** `flatten` для вложенного списка.
2. **Реализовать** `extract_ids` из вложенных dict.
3. **Реализовать** `walk_categories` с отступом по глубине.
4. **Объяснить**, почему наивный Fibonacci — антипример для pipeline (рекурсия vs итерация).
5. **Реализовать** `count_leaves` для `CATEGORY_TREE` (ДЗ).

Задачи Canvas `binary_search`, `max_digit` — [canvas_practice.md](canvas_practice.md); в КТП 8 класса — **модуль 7**, не этот урок.

---

## 5. Профессиональный контекст

JSON API и категории товаров; обход дерева — та же логика, что у узлов decision tree (модуль CV позже).
