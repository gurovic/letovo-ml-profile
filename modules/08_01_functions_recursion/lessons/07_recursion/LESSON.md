## D. Проектирование

### Зачем урок

Вложенные списки и дерево категорий нельзя честно обойти одним «плоским» циклом без дополнительной структуры. Рекурсия — инструмент для таких данных (задел под обход деревьев в ML).

### Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | Рекурсия с **базовым случаем** обходит вложенные списки и деревья `(name, children)` |
| Данные | `NESTED_LIST`, `CATEGORY_TREE` inline; канон — [data/module_datasets.py](../../data/module_datasets.py) |

### Результаты обучения

1. Назвать базовый случай и шаг рекурсии для `flatten`.
2. Реализовать `flatten` для вложенного списка.
3. Реализовать `walk_categories` с отступом по глубине.
4. Объяснить, почему наивный Fibonacci — плохой образец для инженерного кода.

### Профессиональный контекст

JSON API и деревья категорий; тот же приём обхода — у узлов decision tree (позже в профиле).
