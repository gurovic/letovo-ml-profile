## 2. Зачем существует урок

После введения рекурсии нужна серия задач. Заодно закрепляются короткие transform (`lambda` в `map`/`filter`/`sorted`) и сборка шагов в `apply_pipeline` — мост к артефакту `text_stats`.

Новую большую тему не вводим: lambda и pipeline — короткое правило + сразу задачи.

---

## 3. Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | Уже известные приёмы отрабатываются серией задач; *lambda* — короткая запись функции в одну строку; *цепочка шагов* — вызов функций по порядку (`apply_pipeline`) |
| Что поддерживает, но не отвлекает | leaderboard моделей; текстовый mini-pipeline как задел к артефакту |
| Данные урока | `NESTED_API_RESPONSE`, `CATEGORY_TREE`, `EXAM_SCORES`, `MODEL_RUNS`, `FEATURE_ROWS` |

---

## 4. Результаты обучения

1. **Дорешать** рекурсию: `extract_ids` / `walk_categories` / `count_leaves` (что не закрыто на паре 8).
2. **Применить** `map`/`filter`/`sorted(..., key=)` с lambda к спискам данных модуля.
3. **Собрать** leaderboard `MODEL_RUNS` (f1 ↓, при равенстве id ↑).
4. **Реализовать** `apply_pipeline(data, steps)` и прогнать одну строку `FEATURE_ROWS`.
5. **Объяснить** порядок шагов extract → scale → predict.

---

## 5. Профессиональный контекст

`sorted(..., key=)` и pandas `.apply(lambda …)` — один приём. `Pipeline(steps=[...])` в sklearn — тот же смысл, что список функций в `apply_pipeline` (без `fit`).

## 8. Активности и задачи

Серия отработки — **все блоки в одну пару**. Минимум: A1–A2, B1–B2, C1. Остальное — если успевают / дифференциация.

### A. Рекурсия

| # | Задача | Вид | Проверка | Сложность |
|---|---|---|---|---|
| A1 | `extract_ids(NESTED_API_RESPONSE)` | код | список id | базовая |
| A2 | `walk_categories(CATEGORY_TREE)` | код | печать с отступом | базовая |
| A3 | `count_leaves` (если не сделано в ДЗ) | код | число листьев | углублённая |

### B. Lambda

| # | Задача | Вид | Проверка | Сложность |
|---|---|---|---|---|
| B1 | `filter` аномалий на `EXAM_SCORES` (свой порог) | код | список | базовая |
| B2 | Leaderboard `MODEL_RUNS`: f1 ↓, id ↑ | код | порядок id: 25, 30, 305, 101, 200 | базовая |
| B3 | `max(FEATURE_POINTS, key=…)` — farthest point | код | точка с макс. суммой квадратов | углублённая |

### C. Pipeline

| # | Задача | Вид | Проверка | Сложность |
|---|---|---|---|---|
| C1 | `apply_pipeline` + шаги extract → scale → predict на `FEATURE_ROWS[0]` | код | число/предсказание на выходе | базовая |
| C2 | Текстовый pipeline: strip → split → filter коротких токенов | код | список токенов | углублённая |

## 9. Ожидаемые ошибки

| Тип | Что делает / думает учащийся | Что говорит или показывает учитель |
|---|---|---|
| Синтаксис | `sorted(models)` без `key=` | «По чему сортируется словарь по умолчанию?» |
| Pipeline | Scale до извлечения числа из dict | Прогнать типы после каждого шага |
| Pipeline | Шаг возвращает `None` (забыли return) | Печать типа после каждого шага |
| Рекурсия | Путает задачу пары 8 и Canvas binary_search | «binary_search — модуль 7» |
