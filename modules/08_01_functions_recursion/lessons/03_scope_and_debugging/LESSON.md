# Lesson Design: Область видимости и отладка

## 1. Идентификация

| Поле | Значение |
|---|---|
| Модуль | Функции и рекурсия в Python |
| Название урока | Scope и отладка |
| Длительность | 2 академических часа |
| Пререквизиты | Уроки 1–2; функции с return |

---

## 2. Зачем существует урок

Метрика и predict — **отдельные функции**. Ошибка scope или пропущенный `return` даёт ложный вывод «модель хорошая» — типичный инженерный сбой.

---

## 3. Центральная идея

| Поле | Значение |
|---|---|
| Центральная идея | Метрика — функция с контрактом; баг в метрике искажает оценку модели |
| Что поддерживает | accuracy, confusion_counts, журнал отладки |

**Данные:** `PREDICTIONS`, `LABELS` ([data/module_datasets.py](../../data/module_datasets.py))

---

## 4. Результаты обучения

1. **Исправить** `accuracy_buggy` (пропущенный return).
2. **Объяснить** баг global в `update_metrics`.
3. **Найти** баг `predict_pass_buggy` (выражение вместо return).
4. **Реализовать** `confusion_counts` → `(tp, fp, fn, tn)`.
5. **Заполнить** журнал отладки: симптом → причина → исправление.

---

## 5. Профессиональный контекст

В ML-проекте метрики считают отдельно от обучения модели; ошибка в `accuracy` на валидации — риск неверного выбора модели.

---

## 6. Структура урока

| Этап | Цель | Активность |
|---|---|---|
| Эталон | Корректная accuracy | разбор эталонной функции |

| Отладка 3 | predict без return | объяснить None |
