# Unit Planner: динамическое программирование и игры

**Класс:** 8  
**Статус:** план модуля; `LESSON.md` — после финализации группировки пар

## 2. Зачем существует модуль

**Проблема:** наивная рекурсия экспоненциальна; без DP нет инструмента для оптимизации на таблице состояний — типично в логистике и играх.

**Если убрать модуль:** пробел в требованиях Foundation §7; слабая подготовка к олимпиадным и инженерным задачам на перебор состояний.

## 5. Результаты обучения

После модуля учащийся может:

1. **Сравнить** наивную рекурсию и мемоизацию по времени.
2. **Решить** задачу размена монет DP 1D.
3. **Решить** задачу максимума на отрезке DP 1D.
4. **Заполнить** таблицу DP 2D по рекуррентному переходу.
5. **Определить** win/lose позицию в игре с ходами (камни).
6. **Объяснить**, когда DP уместен в инженерии, а когда — избыточен.
