# Unit Planner

Статус: Draft 3

Школьный unit planner (внешняя форма) — [reference/SCHOOL_UNIT_PLANNER.md](../reference/SCHOOL_UNIT_PLANNER.md). Наш шаблон **не копирует** школьную структуру; `UNIT.md` должен содержать данные для экспорта в неё (маппинг — в справочнике).

## Назначение

Unit Planner отвечает на один вопрос:

**Как проектируется модуль?**

Документ заполняется **до** создания уроков, ноутбуков, презентаций и активностей.

Общие правила проектирования материалов — в Material Design Standard. Здесь — **шаблон модуля**.

---

## Как пользоваться

1. Пройти обязательные вопросы из Material Design Standard (раздел 2).
2. Заполнить поля ниже (разделы 1–18).
3. Проверить по чек-листу модуля (раздел 19).
4. Проверить по общему чек-листу Material Design Standard (раздел 11).
5. При экспорте в школьную форму — раздел 20 и [reference/SCHOOL_UNIT_PLANNER.md](../reference/SCHOOL_UNIT_PLANNER.md).

## 11. Последовательность уроков

| # | Урок | Часы | LESSON | Зачем | Центральная идея | Главная активность | Артефакт |
|---|---|---|---|---|---|---|---|
| 1 | | | | | | | |

- **Часы** — академические часы на урок (сумма + итоговая работа = §1).
- **LESSON** — путь к `LESSON.md` (**один урок = одна пара КТП**, 2 ч учёта = **80 мин** проведения). Файл читается учителем как **сценарий пары** (зоны A–C в [Lesson Design](04_LESSON_DESIGN.md)); проектирование — зона D; школьный экспорт — §13 (зона E).
- **Карточка урока** — обязательный `LESSON.md` §13 (все поля школьной строки урока; см. Lesson Design зона E).

Без деталей реализации в UNIT. Каждый урок — отдельный `LESSON.md` по Lesson Design Draft 5.

В таблице для программных тем явно помечать роль урока: **введение** или **отработка**. Отработка — серия задач на уже введённый навык (Foundation §5, Pedagogy §2). Введение и отработку одного навыка — **две строки** таблицы (§11), не один LESSON.

### Длительность

| Компонент | Правило |
|---|---|
| Одна пара | **80 минут** проведения (всегда); 2 академических часа в учёте и §13 |
| Уроки введения | столько, сколько нужно, чтобы показать контракт и 1–2 управляемые задачи |
| Уроки / блоки отработки | обязательны для программных навыков; типично ≥ столько же времени, сколько на введение той же конструкции, либо отдельная пара КТП |
| Итоговая работа | **до 2 уроков**: проектирование/черновик + доработка/сдача; не заменяет первичную отработку |
| Итого | при необходимости **до 10–11 уроков** на модуль (см. КТП класса) |

При нехватке часов — сократить число новых тем, не вырезать отработку.

## 19. Чек-лист модуля

- [ ] модуль имеет ясную цель;
- [ ] каждый урок вносит вклад в цель;
- [ ] ни один урок не существует «по традиции»;
- [ ] пререквизиты достаточны;
- [ ] результаты обучения соответствуют активностям;
- [ ] концептуальная рамка (§7) согласована с уроками и артефактом;
- [ ] метапредметные навыки (§10) привязаны к урокам;
- [ ] инженерная практика присутствует;
- [ ] математика обоснована;
- [ ] итоговый артефакт демонстрирует заявленные результаты;
- [ ] поля §15–18 заполнены или явно отмечены «не применимо»;
- [ ] у каждого урока в §11 заполнена карточка в `LESSON.md` §13;
- [ ] строки [матрицы навыков](../docs/ktp/08_skills_matrix.md) для пар модуля актуальны (см. [07_KTP § Матрица навыков](../docs/07_KTP.md));
- [ ] модуль согласован с траекторией класса (Foundation, раздел 7).

Общий чек-лист материала — Material Design Standard, раздел 11.
