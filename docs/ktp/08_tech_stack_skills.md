## Правила списка

| Правило | Смысл |
|---|---|
| Формулировка | «Уметь …» + в скобках или столбце **конкретный стек** (`matplotlib.pyplot.boxplot`, `sklearn…`) |
| Уровень | То, что реально введено и отработано по КТП 8 класса; не «знать всю библиотеку» |
| Вне scope | Явно помечать **не входит** (9 класс / вне профиля), чтобы не раздувать ожидания |
| Обновление | Вместе с картой инструментов и модулем: новый API в ноутбуках → строка здесь |

**Не писать:** абстрактные «понимание ML» без инструмента; чеклист сдачи артефакта; сюжетные имена датасетов как навык.

| Уметь | Стек | Модуль |

| Посчитать F1 / метрики классификации | `sklearn.metrics.f1_score` (и связанные) | M4 |

## Как пользоваться при проектировании

1. Новый ноутбук 8 класса: навыки из этого списка можно требовать **после** пары введения из [карты инструментов](08.md#карта-инструментов); раньше — нельзя.
2. Входной контроль / 9 класс: этот файл = ожидание «что уже умеют руками».
3. При расхождении с ноутбуком модуля — править **модуль или этот список** в одном цикле; не держать два канона.

**Итого:** чеклист стека после **68** пар КТП 8 класса (4 ч/нед × 34 нед). Классы 9–11 — отдельные файлы по мере появления планов.
