# Notebook Standard

Статус: Draft 2

## Назначение

Notebook Standard отвечает на один вопрос:

**Как проектируется учебный Jupyter-ноутбук?**

Ноутбук — интерактивная среда обучения, а не лекция и не репозиторий кода.

Общие правила материалов — в Material Design Standard. Контекст урока — в Lesson Design. Здесь — **правила формата ноутбука**.

---

## Когда нужен ноутбук

Ноутбук создаётся, если урок требует:

- пошаговых экспериментов;
- итеративной работы с кодом и данными;
- фиксации наблюдений по ходу работы.

Если достаточно скрипта или активности без пошагового сопровождения — ноутбук не обязателен (см. Pedagogy, раздел 11).

---

## 1. Цели формата

Ноутбук помогает учащемуся:

- думать;
- экспериментировать;
- реализовывать;
- исследовать;
- фиксировать выводы.

(Общий принцип «деятельность, а не потребление» — Material Design Standard, раздел 5.)

---

## 2. Структура ноутбука

| # | Раздел | Содержание |
|---|---|---|
| 1 | Название и цель | Одна строка: зачем этот ноутбук |
| 2 | Результаты обучения | Формулировки — по MDS, раздел 3 |
| 3 | Пререквизиты | Что учащийся уже умеет |
| 4 | Импорты | Минимально необходимые |
| 5 | Рабочие блоки | Короткие исполняемые ячейки |
| 6 | Направляемые эксперименты | С изменением параметров |
| 7 | Вопросы для рефлексии | Короткие, по делу |
| 8 | Самостоятельное задание | Расширение, не повторение |
| 9 | Итог | Что усвоено; что дальше |
| 10 | Дополнительно | По желанию, для сильных |

---

## 3. Код

| Правило | Пояснение |
|---|---|
| Короткие ячейки | Одна ячейка — одна мысль |
| Одна цель на ячейку | Не смешивать загрузку, анализ и визуализацию |
| Без магии | Нет необъяснённого кода; каждая функция мотивирована |
| Скрытая сложность | Вспомогательное — в функции или импорте, не в теле урока |
| Воспроизводимость | Ноутбук выполняется сверху вниз без ручных правок |

---

## 4. Текст

- Текст объясняет **решения**, а не синтаксис.
- Предпочитать **вопросы** длинным объяснениям.
- Markdown — для постановки задачи и рефлексии, не для лекции.

---

## 5. Эксперименты

- Учащийся **меняет параметры**, сравнивает результаты, записывает наблюдения.
- Хотя бы один эксперимент — **без заранее заданного ответа** (MDS, раздел 8).
- Ритм: идея → код → наблюдение → объяснение → изменение.

---

## 6. Визуализации

- Каждый график отвечает на **конкретный вопрос**.
- Декоративные графики **не допускаются**.
- Подписи осей и заголовки — осмысленные.

---

## 7. Самостоятельная работа

Каждый ноутбук заканчивается заданием, где учащийся **расширяет** решение, а не повторяет код из ноутбука.

---

## 8. Чек-лист ноутбука

- [ ] выполняется от начала до конца;
- [ ] воспроизводим (фиксированные seed, версии);
- [ ] читаем (короткие ячейки, понятный текст);
- [ ] баланс текста и кода;
- [ ] есть эксперимент с открытым вопросом;
- [ ] есть самостоятельное задание;
- [ ] производит осмысленный артефакт.

Общий чек-лист материала — Material Design Standard, раздел 11.
